# Question 4 — Time Complexity vs Rank $k$

## Goal

Analyze how the time per single SteepestDescent iteration scales with the rank parameter $k$.

## Method

- Run a single SteepestDescent iteration for a range of $k$ values from 1 to $n-1 = 1023$.
- Repeat each measurement 5 times and take the median to reduce noise.
- Plot the results on both linear and log-log scales, and fit a power law $t \propto k^\alpha$.

## Analysis

The dominant operations per iteration are:

| Operation | Complexity |
|-----------|------------|
| Cost: $\operatorname{tr}(M^\top S M)$ via element-wise $M \odot (SM)$ | $O(n^2 k)$ — matrix-matrix product $SM$ |
| Gradient: $2SM$ | $O(n^2 k)$ |
| Riemannian gradient (projection onto tangent space of Sphere) | $O(nk)$ |
| Retraction (normalize) | $O(nk)$ |
| Line search (a few cost evaluations) | $O(n^2 k)$ each |

Here $n = (n-1) = 1023$ is fixed. Since $S \in \mathbb{R}^{(n-1)\times(n-1)}$ is precomputed, the bottleneck is the matrix-matrix product $S M$, which costs $O(n^2 k)$. We therefore expect **linear scaling** in $k$, i.e., $\alpha \approx 1$.

## Output

Two side-by-side plots:
1. **Linear scale:** Time per iteration (ms) vs $k$  
2. **Log-log scale:** With fitted power law to confirm the exponent $\alpha$
